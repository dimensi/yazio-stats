# Дизайн: размеры продуктов, JSON meals, вывод day с разметкой Telegram

**Дата:** 2026-03-05  
**Статус:** утверждён

## Цели

1. Корректно считать и показывать размеры продуктов: при «нестандартной» единице (не г/мл) запрашивать продукт и выводить вес в граммах (вариант A).
2. Формат JSON для `meals --format json`: массив дней с `total`, в каждом item — `amount_display` (русские единицы: г, мл, порц. при fallback).
3. Команда `day` — текст с разметкой Telegram (*жирный*, _курсив_), дата на русском, приёмы пищи с `amount_display` и КБЖУ по приёму.

## 1. Разрешение веса и amount_display

- **Общая утилита** в `src/utils/` (например `productDisplay.ts` или в одном модуле с порциями): по `product_id`, `amount`, `serving` и объекту продукта (если есть) возвращать `{ weightG: number | null, amountDisplay: string }`.
- Правила:
  - `serving` в списке граммовых (`gram`, `g`) → вес = amount, `amount_display` = `"X г"`.
  - `serving` миллилитры (`milliliter`, `ml`) → `amount_display` = `"X мл"`.
  - Иначе: запрос продукта (с кэшем по product_id), в `servings` ищем элемент с `serving === item.serving` → вес = `item.amount * serving.amount` (amount = количество порций), `amount_display` = `"X г"` (всегда вес в граммах при успешном разрешении).
  - Если не нашли или `serving === "-"` и `base_unit === 'g'` → считаем amount уже граммами, `amount_display` = `"X г"`.
  - Fallback при ошибке/неизвестном: `amount_display` с сырой единицей или маппинг в русские (порц., шт., ломт.) без веса.
- Кэш продукта при обходе дня: не дергать API многократно для одного product_id.

## 2. Формат JSON для meals

- Структура: массив дней, каждый с `date`, `total` (calories, protein, carbs, fat), `meals.{breakfast,lunch,dinner,snack}` с `items[]` (product, amount_display) и `nutrients` по приёму.
- `total` считается суммой по `meals.*.nutrients`.
- В item обязательно `amount_display` с русскими единицами; при разрешённом весе — в граммах (`"260 г"`, `"15 мл"`).

## 3. Вывод команды day

- Заголовок и блоки калории/белки/углеводы/жиры в формате как в примере, с разметкой Telegram: `*метка:* значение (N%)`.
- **Дата на русском:** например «4 марта 2026» (без «г.» или с — на усмотрение реализации). Добавить в `src/utils/dates.ts` функцию вывода даты для локали `ru` (например `formatDateLongRu` или параметр локали в `formatDateLong`) и использовать её в текстовом выводе `day`.
- Приёмы пищи: список продуктов с `(amount_display)` в скобках, затем строка _ккал · Б · У · Ж_.
- Использовать общую утилиту для `amount_display`, чтобы везде было единообразно (в т.ч. «X г» при разрешённом весе).

## 4. Синхронизация meals и day

- Убрать отладочные `console.log` из `meals.ts`.
- В `day` при сборке строк по приёмам брать `amount_display` из той же утилиты; рецепты и express — единый стиль (например «— г» или порции по существующей логике).

## 5. Ошибки и границы

- Продукт не найден → fallback: `amount_display` из amount + сырой serving или маппинг в русские единицы.
- Нет подходящего serving в продукте → не считать вес, вывести «X порц.» (или аналог по маппингу единиц).
